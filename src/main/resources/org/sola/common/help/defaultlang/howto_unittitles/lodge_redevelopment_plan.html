Lodge a Redevelopment Plan

A Redevelopment Plan describes changes to a Unit Title Development due to any of the following

    • Subdivision (reduction) of an existing unit or units
    • Enlargement of a unit
    • Erection of one or more new units
    • Removal or one or more existing units

It is a legal requirement that Redevelopment Plans retain the number assigned to the original Standard Unit Development once the redevelopment plan is approved. To satisfy this requirement, and avoid confusion with the original plan, SOLA applies a temporary numeric suffix (e.g. 1, 2, 3, etc.) to the redevelopment plan up until it is approved. Once approved the redevelopment plan is given the original plan number (excluding any suffix) while earlier approved plans for the development are assigned an alphabetic suffix (e.g. A, B, C, etc.)

Steps

    1. Lodge an application with the Record Unit Plan service. You must add a folio from the Unit Title Development to the Property tab of Application Details before you lodge the application. The recommended folio to use is the Common Property folio (e.g. CP/11000).
    2. After lodging the application you will note the Application No. matches the original Unit Title Development Plan Number and includes a numeric suffix (e.g. 11000/2). Note that the first suffix number assigned will be 2 as this will be the second plan lodged for the Unit Title Development.
    3. Ensure you are Assignee for the redevelopment application and open it from your Dashboard.
    4. Start the Record Unit Plan service. The Unit Titles screen will open.
    5. If any existing unit parcels are being enlarged or reduced in size by the redevelopment plan, update their Official Area values on the Unit parcels tab. This applies to all unit parcels whether they are Principal Units, Accessory Units or the Common Property.  SOLA will check that the total area for all unit parcels including the common property matches the area of the underlying parcel.
    6. If any existing unit parcels are being removed by the Redevelopment Plan, select the parcel and click theRemove tool. This will flag the parcel as Delete on Approval and ensure the parcel is removed from the Unit Title Development when the application is approved. Note: SOLA will not let you remove the Common Property parcel. If the Common Property is completely taken by the Redevelpment Plan for other units, set the area of the Common Property to 0 instead.
    7. Use theAdd tool to add any new Principal or Accessory Units to the Unit Title Development. The Add tool will automatically increment the Lot / Parcel Number; however you can edit this value if necessary.
    8. To remove a new unit parcel that was added by mistake, use theRemove tool.
    9. Once you have completed the necessary changes, clickSave. Business rules will run and inform you of any inconsistences or issues with the data. If all business rules pass, close the Record Unit Plan screen.
    10. On the Service tab of the Application Details screen, select the Record Unit Plan Service and click theComplete tool. This will run the SOLA business rules to validate the application data. If there is a critical failure, you must remedy the failure as it will not be possible to approve the application otherwise.
    11. Continue and approve the application using the normal procedures.

Notes

    • When the application is approved, you will notice that the Application No. suffix is removed and the Redevelopment Plan will be assigned the original plan number for the Unit Title Development. If you check the earlier plan applications for the development, you will see they have been assigned an alphabetic suffix with A being the earliest. It is possible for gaps to occur in the alphabetic suffixing in the case where a Redevelopment Plan is cancelled before approval.

Changing Unit Title Folios

Steps

    1. Lodge an application with the Create Unit Titles service. You must add a folio from the Unit Title Development to the Property tab of Application Details before you lodge the application. The recommended folio to use is the Common Property folio (e.g. CP/11000).
    2. If any of the Principal Units have been removed from the development or dissolved into the Common Property, add a Cancel Title service to the application and also add those unit title folios to be cancelled to the Property tab of Application Details.
    3. Ensure you are Assignee for the Create Unit Titles application and open it from your Dashboard.
    4. Start the Create Unit Titles service. The Unit Titles screen will open.
    5. If new units have been added to the Unit Title Development, use theCreate tool on the Strata Properties tab to generate stub folios for the new units. Complete any new Principal Unit Folios as described in the Create Unit Title Folios section.
    6. If any Accessory Units have been added or removed from the development, update the affected Principal Unit folios as required.
    7. Edit the official areas for the Common Property and any Principal Units that have been enlarged or reduced due to the Redevelopment Plan.
    8. Update Unit Entitlements for the Principal Unit folios as required.
    9. If Principal Units need to be removed, use the Cancel Title service toTerminate the unit title folios affected.
    10. Once all changes indicated by the Redevelopment Plan have been processed,Complete the services on the application.
    11. Continue processing the application and use theApprove tool to register all changes to the unit title folios.
